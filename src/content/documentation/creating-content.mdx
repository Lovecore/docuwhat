# Creating Content in DocuWhat

Creating content is as simple as creating a Markdown file. This guide covers everything you need to know.

## File Types

DocuWhat supports two file types:

### 1. Markdown Files (`.md`)
Standard Markdown files for simple content:

<CodeBlock language="markdown">
---
title: Simple Page
---

# Simple Page

This is standard **Markdown** content.
</CodeBlock>

### 2. MDX Files (`.mdx`)
Markdown with JSX components for rich content:

```yaml
---
title: Rich Page
---
```

Then in your MDX content, you can use components like `VideoPlayer` with the appropriate props.

<Callout type="info" title="Recommendation">
  Use `.mdx` files for most content - they support everything `.md` files do, plus components.
</Callout>

## Frontmatter Reference

Every file should start with frontmatter:

<CodeBlock language="yaml">
---
title: "Page Title"           # Required: Shows in navigation
description: "Brief summary"   # Optional: For search & previews  
tags: ["tag1", "tag2"]        # Optional: Improves searchability
order: 1                      # Optional: Navigation sort order
video: "/videos/intro.mp4"    # Optional: Featured video
updated: "2024-01-15"         # Optional: Last update date
icon: "ðŸš€"                    # Optional: Icon for navigation
---
</CodeBlock>

### Frontmatter Fields Explained

| Field | Type | Purpose | Example |
|-------|------|---------|---------|
| `title` | string | Page title in navigation & browser | `"Getting Started"` |
| `description` | string | Search preview & SEO | `"Learn the basics"` |
| `tags` | array | Categorization & search | `["tutorial", "basic"]` |
| `order` | number | Sort position in nav | `1`, `2`, `10` |
| `video` | string | Path to featured video | `"/videos/demo.mp4"` |
| `updated` | string | Last update date | `"2024-01-15"` |
| `icon` | string | Navigation icon | `"ðŸ“š"`, `"ðŸ”§"` |

## Content Structure

### Basic Markdown

All standard Markdown works:

<CodeBlock language="markdown">
# Heading 1
## Heading 2  
### Heading 3

**Bold text** and *italic text* and `inline code`

- Unordered list item 1
- Unordered list item 2
  - Nested item

1. Ordered list item 1
2. Ordered list item 2

> Blockquote for important notes

[Link text](https://example.com)

![Image alt text](/images/example.png)
</CodeBlock>

### Code Blocks

Use triple backticks with language specification:

<CodeBlock language="markdown">
```javascript
function hello() {
  console.log("Hello, DocuWhat!");
}
</CodeBlock>

```python
def hello():
    print("Hello, DocuWhat!")
```
</CodeBlock>

## Creating Your First Page

### Step 1: Choose Location

Decide where your file should live:

<FileTree data={[
  {
    name: "src/content/",
    type: "folder",
    children: [
      { name: "your-page.mdx", type: "file" },
      {
        name: "category/",
        type: "folder",
        children: [
          { name: "nested-page.mdx", type: "file" }
        ]
      }
    ]
  }
]} />

### Step 2: Create File

Create a new `.mdx` file:

<CodeBlock language="bash">
touch src/content/guides/my-guide.mdx
</CodeBlock>

### Step 3: Add Content

```yaml
---
title: My First Guide
description: Learning to use DocuWhat
tags: ["guide", "tutorial"]
order: 1
---
</CodeBlock>

Then write your content using:
- Markdown headings and formatting
- MDX components like Callout for important information
- Lists and structured content
- Clear sections and conclusions

### Step 4: View Your Page

Save the file and navigate to `/guides/my-guide` in your browser. The page is instantly available!

## Live Editing

Changes are reflected immediately in development:

1. Edit your `.mdx` file
2. Save the file
3. Browser auto-refreshes with changes

No build step needed during development!

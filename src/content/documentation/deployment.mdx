# Deployment Guide

Deploy your DocuWhat documentation to production with these popular hosting platforms.

## Pre-Deployment Checklist

<Callout type="info" title="Before Deploying">
  Ensure your documentation is production-ready by completing this checklist.
</Callout>

- [ ] All content reviewed and edited
- [ ] Images and videos optimized
- [ ] Search functionality tested
- [ ] Navigation structure finalized
- [ ] Dark mode tested
- [ ] Mobile responsiveness verified
- [ ] Build runs without errors
- [ ] Environment variables configured

## Build for Production

First, test the production build locally:

<CodeBlock language="bash">
# Install dependencies
npm install

# Build for production
npm run build

# Test production build
npm run start
</CodeBlock>

Visit `http://localhost:3000` to test the production build.

## Deployment Options

### Option 1: Vercel (Recommended)

Vercel offers the best Next.js experience with zero configuration.

#### Method A: Git Integration

1. Push your code to GitHub/GitLab/Bitbucket
2. Visit [vercel.com](https://vercel.com)
3. Click "New Project"
4. Import your repository
5. Deploy with default settings

#### Method B: Vercel CLI

<CodeBlock language="bash">
# Install Vercel CLI
npm i -g vercel

# Deploy
vercel

# Follow prompts to complete deployment
</CodeBlock>

**Vercel Features:**
- Automatic HTTPS
- Global CDN
- Preview deployments
- Analytics
- Web Vitals monitoring

### Option 2: Netlify

Netlify provides excellent static hosting with simple deployment.

#### Method A: Drag & Drop

1. Build your project: `npm run build`
2. Visit [app.netlify.com](https://app.netlify.com)
3. Drag the `.next` folder to deploy

#### Method B: Git Integration

1. Push to GitHub/GitLab/Bitbucket
2. Connect repository in Netlify
3. Configure build settings:

<CodeBlock language="yaml">
Build command: npm run build
Publish directory: .next
</CodeBlock>

#### Method C: Netlify CLI

<CodeBlock language="bash">
# Install Netlify CLI
npm i -g netlify-cli

# Build project
npm run build

# Deploy
netlify deploy --prod --dir=.next
</CodeBlock>

### Option 3: GitHub Pages

Deploy directly from your GitHub repository.

<Callout type="warning">
  GitHub Pages requires additional configuration for Next.js apps with dynamic routes.
</Callout>

1. Add deployment script to `package.json`:

<CodeBlock language="json">
{
  "scripts": {
    "deploy": "next build && next export && touch out/.nojekyll",
    "deploy:gh": "gh-pages -d out -t true"
  }
}
</CodeBlock>

2. Install gh-pages:

<CodeBlock language="bash">
npm install --save-dev gh-pages
</CodeBlock>

3. Update `next.config.mjs`:

<CodeBlock language="javascript">
const nextConfig = {
  output: 'export',
  basePath: '/your-repo-name',
  images: {
    unoptimized: true,
  },
};
</CodeBlock>

4. Deploy:

<CodeBlock language="bash">
npm run deploy
npm run deploy:gh
</CodeBlock>

### Option 4: Self-Hosted

Deploy to your own server with Node.js.

#### Using PM2

<CodeBlock language="bash">
# On your server
git clone your-repo
cd your-repo
npm install
npm run build

# Install PM2
npm install -g pm2

# Start with PM2
pm2 start npm --name "docuwhat" -- start

# Save PM2 configuration
pm2 save
pm2 startup
</CodeBlock>

#### Using Docker

Create `Dockerfile`:

<CodeBlock language="dockerfile">
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:18-alpine AS runner
WORKDIR /app
COPY --from=builder /app/.next ./.next
COPY --from=builder /app/public ./public
COPY --from=builder /app/package*.json ./
COPY --from=builder /app/src/content ./src/content
RUN npm ci --production

EXPOSE 3000
CMD ["npm", "start"]
</CodeBlock>

Build and run:

<CodeBlock language="bash">
# Build Docker image
docker build -t docuwhat .

# Run container
docker run -p 3000:3000 docuwhat
</CodeBlock>

### Option 5: AWS Amplify

Deploy with AWS Amplify for enterprise scaling.

1. Install Amplify CLI:

<CodeBlock language="bash">
npm install -g @aws-amplify/cli
amplify configure
</CodeBlock>

2. Initialize Amplify:

<CodeBlock language="bash">
amplify init
amplify add hosting
</CodeBlock>

3. Deploy:

<CodeBlock language="bash">
amplify publish
</CodeBlock>

## Environment Variables

Configure environment variables for production:

<CodeBlock language="bash">
# .env.production
NEXT_PUBLIC_SITE_URL=https://your-domain.com
NEXT_PUBLIC_ANALYTICS_ID=your-analytics-id
</CodeBlock>

**Platform-Specific Configuration:**

- **Vercel**: Set in dashboard → Settings → Environment Variables
- **Netlify**: Set in Site Settings → Environment Variables
- **Docker**: Use `docker run -e VAR=value`
- **PM2**: Use ecosystem file

## Custom Domain

### Vercel
1. Go to project settings
2. Add domain in "Domains" section
3. Update DNS records

### Netlify
1. Site Settings → Domain Management
2. Add custom domain
3. Configure DNS

### DNS Configuration

Add these records to your domain:

<CodeBlock language="text">
Type: A
Name: @
Value: [Platform IP]

Type: CNAME
Name: www
Value: [Platform domain]
</CodeBlock>

## Performance Optimization

### Image Optimization

Use Next.js Image component:

<CodeBlock language="jsx">
import Image from 'next/image';

<Image
  src="/image.jpg"
  alt="Description"
  width={800}
  height={400}
  loading="lazy"
/>
</CodeBlock>

### Video Optimization

For large videos, use external CDN:

<CodeBlock language="mdx">
<VideoPlayer src="https://cdn.example.com/video.mp4" />
</CodeBlock>

### Caching Strategy

Configure caching headers in `next.config.mjs`:

<CodeBlock language="javascript">
module.exports = {
  async headers() {
    return [
      {
        source: '/:all*(svg|jpg|jpeg|png|gif|ico|webp)',
        headers: [
          {
            key: 'Cache-Control',
            value: 'public, max-age=31536000, immutable',
          },
        ],
      },
    ];
  },
};
</CodeBlock>

## Monitoring

### Analytics

Add analytics to track usage:

<CodeBlock language="jsx">
// src/app/layout.tsx
import { Analytics } from '@vercel/analytics/react';

export default function Layout({ children }) {
  return (
    <>
      {children}
      <Analytics />
    </>
  );
}
</CodeBlock>

### Error Tracking

Integrate error tracking:

<CodeBlock language="bash">
npm install @sentry/nextjs
</CodeBlock>

Configure Sentry:

<CodeBlock language="javascript">
// sentry.client.config.js
import * as Sentry from '@sentry/nextjs';

Sentry.init({
  dsn: process.env.NEXT_PUBLIC_SENTRY_DSN,
  environment: process.env.NODE_ENV,
});
</CodeBlock>

## Continuous Deployment

### GitHub Actions

Create `.github/workflows/deploy.yml`:

<CodeBlock language="yaml">
name: Deploy

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: 18
      - run: npm ci
      - run: npm run build
      - run: npm run test
      # Add deployment step for your platform
</CodeBlock>

## Troubleshooting

### Common Issues

**Build Failures:**
- Check Node.js version (18+ required)
- Clear cache: `rm -rf .next node_modules`
- Reinstall dependencies: `npm install`

**404 Errors:**
- Verify build output includes all pages
- Check `next.config.mjs` for proper configuration
- Ensure content files are included in deployment

**Slow Performance:**
- Enable compression
- Optimize images and videos
- Use CDN for static assets
- Implement caching strategies

## Security Considerations

<Callout type="warning" title="Security Best Practices">
  Always follow security best practices for production deployments.
</Callout>

1. **Use HTTPS** - Always serve over HTTPS
2. **Set Security Headers** - Configure CSP, HSTS, etc.
3. **Environment Variables** - Never commit secrets
4. **Regular Updates** - Keep dependencies updated
5. **Access Control** - Implement authentication if needed
